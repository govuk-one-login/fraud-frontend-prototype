MVP+ View

We've drafted an initial MVP journey based on early user insights, requirements and constraints. We will continue to test this as well as post MVP functioanlity that is emerging from user needs. The MVP functioanlity stays true to the strategic automated solution, allowing user to download (receive) and add (transmit) fraud shared signals.

View MVP+ view journey

Iterated event pages

Supporting hint content was added to the events that users struggled to understand. Also looked to mage the intervention question be more passive, as the question doesn't neccassarily trigger an intervention.

View reason page

Iterate add journey

In this version we explored allowing users to add event details before adding user IDs.

An assumption is that user will have more of an understanding that they are adding IDs to a specific event and if they need to add more events to IDs they would do that in a new action.

Upload single

Earlier on in the design we explored adding user IDs singularly, however it didn't test well. After a design crits session I was reminded of this approach might have merit in reducing cognitive load.

Confirmation screen

Updated the confirmation screen to give user information around next steps and time scales.

View confirmation

Download confirm

Explored use of checkbox to avoid the use of the assitional kick out screen. Users understood the approach but did express an expectation that selecting no would return the user back to the task screen.

Time stamp

Explored use of time stamp... still more work to do. I wonder if a simple notification would be more useful. We will lok to test this and time stamp to understand its value, and answer the user need around understainging when new information has been added.

Multiple submission handling MVP

An unhappy path was realised, that needed addressing. When users submit fraud information that matches a previously submitted fraud shared signal, we need to mitigate the risk around duplication of data stored. We will look to test this approach with users in coming research sessions.

View handling of multiple submissions.
