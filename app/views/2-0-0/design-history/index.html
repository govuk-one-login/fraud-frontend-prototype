This is the design story for the Shared Signals UI prototype {% include('../includes/version.html') %}.

The main features of this initial prototype include:

{% include('../includes/summary.html') %}

Event visability

Based on early insights it appears users are event focused, and only interested in specific events. initial designs hid functionality and users had to look to find the releavant information.

This impacts both downloading information and viewing information.

Downloading information

Simplified the download journey by removing the are you sure screen.

This meant leading with the reasons / events screen, and then asking users to select relevant download in the next step.

Viewing information

For early testing of this functionality we moved away from 3 "filter options" (last 30 days, since log in and all), instead leading with the specific events selected. We hope to identify the additional required views in research.

This has resulted in users gettingf to view the information they want to quicker, and in less steps.

We also removed rationale codes and replaced them with interventions, as that information will be available to API users and flagged as useful to UI users.

Upload information

We iterated the upload screen based on early feedback, by including more of the reasons on the same screen and also added secondry pages for the more complex reasons.
