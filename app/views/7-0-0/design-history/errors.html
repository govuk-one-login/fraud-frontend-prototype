Error screens

Background

For SSF-UI MVP, error screens had not been designed. This piece of work introduces some unhappy path screens into the shared signals UI.

Error screens included as part of this work include:

  • A 500 error, if there is an unexpected problem with the service, such as a server issue
  • A 404 error, if the user enters a URL which does not exist
  • A problem when a user is submitting fraud information
  • A problem when a user is downloading fraud information
  • Improvements to the downnload jounrney flow, so that users are aware of how many records there are available to download, or if there are no records available to download

To do

  • Content review of all error screens
  • Use user research to validate our expectation that fraud analysts should only be able to download a file if we have records available to download.
