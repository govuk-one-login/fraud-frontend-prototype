Version {% include('../includes/version.html') %}

This is the design story for the Shared Signals UI prototype {% include('../includes/version.html') %}.

This iteration contains some significant changes, based on early insight from internal users. The main features of this initial prototype include:

{% include('../includes/summary.html') %}

Removed download journey

Users mentioned wanting to view the information before downloading. The call to action to download seemed redundanct and added to the cognitive load on the task screen.

One user mentioned wanting to see a "homepage" showing links to past submissions. We will wait to see if a pattern emmerges around this need.

View updated task screen.

Added sort and filter to view screens

Users wanted to get to the list page sooner and some users mentioned being user led, so seeing lists and filters on previous screens was jarring.

Filters and sortable tables were implemented, however for research we may look to add one interactive element at a time. In addition we have renamed Intervention with status and can make use of the GDS tags pattern, but will first do some research to see if that is needed and some card sorting to understand how colours land with users. View information lists with filters.

A javascript disable approach was also maintained.

Search

Search was mentioned by both stakeholders and internal users. We created an initial search pattern. View search functionality.

Added ID details view

Users expected to see more information than what was on the list page. Some audit history and a chronological view of the data was mentioned. The priority of the information is to be explored in the next round of research. View user id details.

Iterated upload journey

We made a number of changes to the upload journey including with updated event names, and additional optional fields here. We also simplified the journey by removing the question around "adding more updates" during the same session. This question caused some confusion, as users expected to be able to add additional information.

MoJ frontend and Jquery added

MoJ frontend and Jquery were added to the prototype to allow for filtering and sorting

Service name

With more focus on viewing information over downloading information, the temporary service name will need exploring in the next round of research.
