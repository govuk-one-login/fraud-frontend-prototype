Fraud Admin UI MVP

The Fraud admin UI was designed by Barry Khan (who left GDS in January 2024). There was less documented about how the Fraud admin interface design was iterated, compared to the Shared Signals UI.

The main features of Fraud Admin UI MVP included:

  • Applying a manual account intervention
  • Hashing email addresses and telephone numbers

Removal of hashing during dev build:

During dev build of the Fraud admin interface, the hashing feature was de-scoped due to delivery timeframes. This is to be brought back into the UI as an MVP+ feature. To view the prototype with this feature, click 'set up prototype' and select the 'request hashed information' radio.

Changes to navigation during dev build:

During dev build the navigation header design was changed without involvement from UCD. The back links on the 'manual account interventions details' screen and 'what is the email address / telephone number screen' were removed, and replaced with a 'home' navigation item in the header to take users back to the landing screen. The navigation has since been changed to reflect the dev build, however there may not be a need to have a secondary navigation header in the design at all. Consider removing this as part of MVP+.

Content review of account interventions

Melanie Ring did a great job at 2i'ing the content for the account interventions screens. The content is now reflected in the designs. Her design rationales are wirtten up in this document.

  • Challenge the need for the header navigation in the UI - this has been built in the design but may not be the simplest way to navigate in the UI.
